# QQ农场计时器 - 问题排查报告

## 用户反馈的三个问题

### 1. 语音输入无法使用
**可能原因：**
- 页面失焦时自动停止语音识别
- 麦克风权限问题
- 浏览器不支持Web Speech API

**已确认问题：**
- ✅ visibilitychange事件在页面隐藏时会停止语音识别
- ✅ 停止时设置stopReason为'manual-stop'，会显示"已停止聆听"
- ✅ 用户点击后可能还没来得及说话就被停止了

### 2. 作物时间问题（尤其是第二季的收获时间）
**已确认逻辑：**
- calcTotalGrowTime: `firstTime * m + reTime * m * (seasons - 1)`
- getSeasonTimes: 返回 `[首季时间, 第二季时间, ...]`
- triggerAlarm创建下一季: 使用`timer.seasonTimes[nextSeason - 1]`

**可能问题：**
- 需要确认seasonTimes是否正确传递到timer对象
- 需要确认计算逻辑是否有边界情况

### 3. 后台闹钟触发没有收到任何通知
**已确认问题：**
- ⚠️ **Service Worker没有实现showNotification接口！**
- 代码调用了`registration.showNotification()`，但sw.js中没有处理
- visibilitychange的else分支会立即停止语音识别，这可能导致用户没说话就被停止

## 排查链路

### 通知链路（前台触发）
```
triggerAlarm()
  └─> sendBrowserAlarmNotification()
       ├─> 检查权限 Notification.permission
       ├─> 获取Service Worker注册
       ├─> 调用 registration.showNotification()  ❌ sw.js未实现！
       └─> 失败后显示toast
```

### 通知链路（后台触发）
```
visibilitychange (!document.hidden)
  └─> 检查过期闹钟
       └─> triggerAlarm()  [同上]
  └─> cleanupExpiredAlerts()
```

1. **在sw.js中实现notification相关接口**
   - 实现showNotification方法
   - 处理notificationclick事件（已有）

2. **优化语音识别的自动停止逻辑**
   - 不要在页面失焦时立即停止
   - 只在检测到失焦超过一定时间后才停止
   - 或者给用户一个明确的停止提示

### 优先级P1（体验优化）
3. **验证多季作物时间计算**
   - 检查startPlantTimer是否正确传递seasonTimes
   - 添加调试日志验证计算结果
   - 确保第二季时间正确

4. **增强调试能力**

self.addEventListener('message', (event) => {
  if (event.data && event.data.type === 'SHOW_NOTIFICATION') {
    const { title, options } = event.data;
    self.registration.showNotification(title, options);
  }
  if (event.data && event.data.type === 'SKIP_WAITING') {

  }
});
