# QQ农场两季作物第二季成熟时间分析报告

## 2026年最新官方规则（来自搜索结果）

### 核心机制：第二季成熟时间严格减半

根据2026年2-3月的多个权威来源确认（QQ频道、百度贴吧玩家实测）：

**规则说明：**
- **第一季（一熟）**：正常完整生长，时间为 firstTime
- **第二季（二熟）**：自动跳过两个生长阶段，成熟时间直接为第一季的**一半**（firstTime / 2）
- **收益**：第二季仍获得100%的果实和经验，不减半

**来源证据：**
1. QQ频道【数据党必看】二季作物深度解密（2026-02-28）：
   > "第二季：自动跳过两个生长阶段，成熟时间直接减半，经验、产量保持不变"

2. 百度贴吧玩家实测（2026-03-30）：
   > "61级后的两季作物第二季成熟时间减半，而且还能再施一次普肥"

3. 计算示例（蘑菇）：
   - 首季：4小时
   - 第二季：2小时（4 / 2 = 2）
   - 总时长：6小时
   - 收益：用 1.5 倍时间，获得 200% 收益

## 当前计时器实现对比

### 1. 数据结构（plants-data.js）

#### 内置作物数据（问题所在）
当前内置作物数据中的两季作物，其 `reTime` 字段**全部等于 firstTime**，而非 firstTime / 2：

```javascript
"蘑菇": { name: "蘑菇", firstTime: 4, reTime: 4, seasons: 2, ... }  // ❌ 错误
"菠萝": { name: "菠萝", firstTime: 8, reTime: 8, seasons: 2, ... }  // ❌ 错误
"椰子": { name: "椰子", firstTime: 4, reTime: 4, seasons: 2, ... }  // ❌ 错误
// ... 所有 2 季作物都是如此
```

**影响范围：**
检查到至少 40+ 种两季作物的 `reTime` 设置错误，包括：
- 蘑菇（4小时，应第二季2小时）
- 菠萝（8小时，应第二季4小时）
- 椰子（4小时，应第二季2小时）
- 花生（8小时，应第二季4小时）
- ... 等等

#### 自定义植物处理（正确）
`normalizePlantRecord` 函数中，对于自定义多季作物，计算逻辑是正确的：

```javascript
if (plant.seasons > 1) {
    const firstTime = Number(plant.firstTime) || 0;
    plant.reTime = Math.round(firstTime / 2 * 10) / 10;  // ✅ 正确：reTime = firstTime / 2
}
```

### 2. 计算函数分析

#### calcTotalGrowTime - 总成熟时间
```javascript
function calcTotalGrowTime(plantName, landType) {
    const plant = PLANTS_DATABASE[plantName];
    const land = LAND_TYPES[landType];
    if (!plant || !land) return 0;
    const m = 1 - land.timeBonus;
    let total = plant.firstTime * m;
    if (plant.seasons > 1) total += plant.reTime * m * (plant.seasons - 1);
    return Math.round(total * 10) / 10;
}
```

**影响：**
- 由于内置作物 `reTime = firstTime`，导致总时间计算为 `firstTime * seasons`（错误）
- 正确应为 `firstTime + firstTime/2 = 1.5 * firstTime`

#### getSeasonTimes - 分季时间数组
```javascript
function getSeasonTimes(plantName, landType) {
    const plant = PLANTS_DATABASE[plantName];
    const land = LAND_TYPES[landType];
    if (!plant || !land) return [];
    const m = 1 - land.timeBonus;
    const times = [Math.round(plant.firstTime * m * 10) / 10];
    for (let i = 1; i < plant.seasons; i++) times.push(Math.round(plant.reTime * m * 10) / 10);
    return times;
}
```

**影响：**
- 返回的季数时间数组也是错误的
- 例如蘑菇金土地（0.2加成）：
  - 错误：[3.2, 3.2]（首季3.2h，第二季3.2h）
  - 应为：[3.2, 1.6]（首季3.2h，第二季1.6h）

## 具体问题案例

### 案例：蘑菇在金土地种植

#### 当前实现（错误）
```javascript
// 数据
firstTime: 4, reTime: 4, seasons: 2
land: gold, timeBonus: 0.2

// 计算
首季时间 = 4 * (1 - 0.2) = 3.2 小时
第二季时间 = 4 * (1 - 0.2) = 3.2 小时
总时间 = 6.4 小时
```

#### 应为正确实现
```javascript
// 数据
firstTime: 4, reTime: 2, seasons: 2
land: gold, timeBonus: 0.2

// 计算
首季时间 = 4 * (1 - 0.2) = 3.2 小时
第二季时间 = 2 * (1 - 0.2) = 1.6 小时
总时间 = 4.8 小时
```

**时间差异：6.4小时 vs 4.8小时（差1.6小时）**

### 方案2：在 normalizePlantRecord 中强制修正

```javascript
function normalizePlantRecord(plant, { isCustom = false } = {}) {
    if (!plant) return plant;
    // ... 原有代码 ...

    // 强制修正内置两季作物的 reTime
    if (plant.seasons > 1 && !plant.isCustom) {

}
```

**缺点：** 需要在加载后统一处理，可能影响性能

## 影响评估

### 功能影响
1. **闹钟时间错误**：两季作物的第二季闹钟时间晚于实际游戏
2. **收益计算错误**：方案推荐中的效率分析不准确
3. **总时长显示错误**：卡片上的"X季·总XXh"时间偏长

### 用户体验影响
- 按照计时器提醒，用户会错过第二季实际成熟时间
- 可能导致用户对计时器产生不信任
